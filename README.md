# Mini Git

Mini Git은 Git의 핵심 자료구조인 DAG(Directed Acyclic Graph)와 해시 매핑, 그리고 역색인(Inverted Index), 위상 정렬(Topological Sort), 최단 경로 탐색, 3방향 병합(3-Way Merge) 등의 알고리즘을 직접 구현하여 시뮬레이션하는 CLI 기반 버전 관리 애플리케이션입니다.

본 프로젝트는 리팩토링을 통해 **모듈화(Separation of Concerns)**가 완료되었으며, 사용자 대화형 CLI는 Python 표준 라이브러리인 `cmd.Cmd` 기반의 쉘 구조로 재구성되어 사용자 편의성과 명령어 파싱 안전성이 대폭 향상되었습니다.

## 3. 실행 및 테스트 방법
프로젝트 루트 디렉토리에서 Python 3.10 이상을 사용하여 실행합니다.

### 3.1 CLI (대화형 쉘) 실행
```bash
python3 main.py
```
* `cmd.Cmd` 기반의 대화형 프롬프트(`mini-git> `)가 제공됩니다.
* 입력 명령어의 대소문자를 구분하지 않으며, `Ctrl+D`(EOF) 혹은 `EXIT`, `QUIT`을 통해 안전하게 종료할 수 있습니다.
* `HELP <command>`를 통해 특정 명령어의 상세 설명과 사용법을 조회할 수 있습니다.

### 3.2 통합 테스트 스위트 실행
대용량 스트레스/성능 테스트를 포함한 기능 명세 요건을 단일 명령어로 일괄 검증합니다.
```bash
python3 -m unittest test_minigit.py
```

## 4. 주요 기능 및 구현 특징

### 4.1 커밋 그래프 (DAG) 및 불변 노드
* 모든 커밋은 0개 이상의 부모 커밋을 가질 수 있는 방향성 비순환 그래프(DAG) 구조를 가집니다.
* 데이터 무결성을 위해 `Commit` 객체는 파이썬 `@dataclass(frozen=True)`로 설계되어 불변성(Immutability)을 보장합니다.
* 커밋 해시(Hash)는 디버깅과 가독성을 위해 순차 정수 카운터를 문자열화한 값(`"1"`, `"2"`, `"3"`)을 해시값으로 대용하여 사용합니다.
* 모든 커밋 노드는 빠른 임의 조회를 위해 Python 해시 테이블 구조인 `commit_dict`에 적재됩니다.

### 4.2 역색인 (Inverted Index)
* 대량의 커밋 히스토리 내에서 메시지 검색 성능을 비약적으로 높이기 위해 검색엔진식 역색인을 사용합니다.
* **단어 역색인 (`commit_word_dict`)**: 커밋 메시지를 소문자로 표준화하고 토큰화하여 각 단어 키워드가 출현한 커밋 ID 목록을 고속 조회할 수 있게 맵을 구성합니다.
* **작성자 역색인 (`commit_author_dict`)**: 소문자로 정규화된 작성자명을 기준으로 해당 작성자의 커밋 리스트를 매핑합니다.
* 선형 전체 검색($O(N)$) 대비 상수 시간($O(1)$)에 수렴하는 조회를 지원합니다.

### 4.3 위상 정렬 (Topological Sort)
* `LOG` 명령어 호출 시 단순 선형 출력이 아니라, 부모 커밋이 자식 커밋보다 항상 먼저(과거에서 미래 순으로 지목될 수 있게) 출력되는 위상 정렬 출력을 지원합니다.
* Kahn의 진입 차수(In-degree) 감소 알고리즘을 활용하여 완벽한 시간적 선후 인과관계를 나열합니다.

### 4.4 최단 경로 탐색 (`PATH`)
* 두 커밋 해시 사이의 최단 경로(최소 간선 개수)를 BFS(너비 우선 탐색)로 해결하며, 커밋의 상하 흐름(부모/자식)을 무방향(Undirected) 간선으로 취급하여 질의합니다.
* 자식 노드 방향 탐색 시 캐싱된 `children_dict`를 사용하여 조회를 최적화하였습니다.
* 다중 최단 경로가 존재할 시, 경로 해시를 연결한 문자열의 **사전순(Lexicographical) 최소 경로**를 선택(Tie-breaking)하도록 설계되어 결정론적인 출력을 제공합니다.

### 4.5 사용자 정의 정렬 알고리즘 (병합 정렬)
* `LOG --sort-by=date|author` 정렬 시 파이썬 내장 정렬 API(`sorted()`)의 도움 없이 직접 구현한 **병합 정렬(Merge Sort)**을 구동합니다.
* 병합 정렬은 비교 정렬 알고리즘의 한계선인 최악/평균 시간복잡도 $O(N \log N)$을 만족하며 동점 조건일 때 원래의 순서를 무너뜨리지 않는 **안정 정렬(Stable Sort)**의 명세를 만족합니다.

### 4.6 조상 탐색 (명시적 스택 기반 DFS)
* 특정 커밋에서 도달 가능한 모든 과거 이력(조상 커밋)을 깊이 우선 탐색(DFS)으로 탐색합니다.
* 대규모 커밋 그래프 환경에서 파이썬의 기본 호출 스택 제한(~1,000회)으로 인한 Stack Overflow 에러를 완벽히 격리하기 위해, 명시적으로 힙 영역의 리스트 자료구조를 스택으로 다루는 **반복적 DFS(Iterative DFS)**로 안전하게 조상을 탐색합니다.
* 의미론적으로 올바른 집합 자료구조(Set ADT)를 반환하여 중복을 차단하고 최적 성능을 보장합니다.

### 4.7 3방향 병합 (3-Way Merge) 및 공통 조상 (LCA)
* 두 갈래로 나뉘어 변경되던 브랜치를 지능적으로 병합하기 위해 **3-Way Merge** 알고리즘을 사용합니다.
* 두 브랜치 끝의 가장 가까운 공통 조상(Lowest Common Ancestor, LCA)을 역방향 BFS로 검출하고 이를 Base 스냅샷으로 지정합니다.
* LCA 탐색 시 `parent_map` 부모 포인터를 기록하여 LCA 발견 시 단 1회 역추적하여 경로를 복구함으로써 시간 및 공간 복잡도를 $O(V)$로 최적화했습니다.
* Base, HEAD(Mine), Remote(Theirs) 세 버전을 비교 대조하여 한쪽 브랜치만 수정한 부분은 자동으로 취합하고, 양쪽 모두 다르게 변경한 충돌 상황에서는 `<<<<<<< HEAD`와 같은 충돌 지시 마커를 파일 내부에 명시하여 사용자의 수동 해결을 지시합니다.

### 4.8 REPL 어휘 분석 (Lexical Analysis)
* 터미널 프롬프트에서 `shlex.split()` 모듈을 활용하여 사용자 명령어를 어휘 분석(Lexing) 처리합니다.
* 이를 통해 큰따옴표나 작은따옴표로 감싸진 공백을 포함하는 커밋 메시지 등의 인자가 단일 문자열 토큰으로 올바르게 보존됩니다.

### 4.9 REPL 편의 기능 및 세션 안정성 예외 처리
* **명령어 대소문자 구분 배제**: `precmd` 메서드를 재정의하여 입력 명령어의 첫 토큰을 대문자로 자동 치환함으로써 사용자의 타이핑 편의성을 높였습니다.
* **비정상 종료 방지 (Crash-safe)**: 개별 명령어 실행 메서드(`do_*`) 레벨에서 `ValueError`, `MergeConflictError` 등의 커스텀/표준 예외를 `try-except` 블록으로 포착하여 에러 상황에서도 세션이 강제 종료되지 않고 대화형 프롬프트가 유지되도록 예외 복구력을 높였습니다.
* **Ctrl+D (EOF) 안정적 처리**: 사용자가 프롬프트에서 `Ctrl+D`를 눌러 나갈 때 개행 문자를 깔끔하게 처리한 뒤 CLI를 안전하게 종료하도록 `do_EOF` UX 보완 코드를 구현했습니다.
* **데이터 정렬의 표준화**: `USERLIST` 및 `BRANCH` 명령어 조회 시, 내장 정렬 라이브러리 없이 자체 구현한 퀵 정렬 기반 `custom_sort` 메서드를 통해 목록을 항상 사전순으로 정렬하여 일관된 뷰를 제공합니다.

### 4.10 통합 성능 및 스트레스 검증 테스트
* `test_minigit.py`에 100회 이상의 대규모 가상 커밋 생성 및 무작위 분기/병합을 유발하는 스트레스 시나리오를 설계했습니다.
* 대규모 그래프 구조 내에서 무방향 BFS 최단 경로 탐색, 역색인 키워드 조회, 병합 정렬 알고리즘의 동작 성능 및 데이터 무결성을 엄격하게 자동 검증합니다.

## 5. 명령어 일람 (Command Reference)

| 명령어 | 형식 | 설명 |
| :--- | :--- | :--- |
| **HELP** | `HELP [command]` | 도움말 목록을 보거나 특정 명령어의 상세 사용법을 조회합니다. |
| **INIT** | `INIT <user_name>` | 저장소를 초기화하고 기본 `main` 브랜치 설정 및 작성자를 등록합니다. |
| **BRANCH** | `BRANCH [branch_name]` | 인자 생략 시 브랜치 목록을 조회하고(현재 브랜치는 * 표시), 인자 입력 시 현재 HEAD 커밋을 가리키는 새로운 브랜치를 생성합니다. |
| **SWITCH** | `SWITCH <branch_name>` | 활성화된 브랜치를 변경합니다. (HEAD 이동) |
| **COMMIT** | `COMMIT <message> [file_name:content ...]` | 현재 브랜치가 가리키던 커밋을 부모로 두는 신규 커밋을 생성(파일 스냅샷 file_meta 반영)하고 역색인을 갱신합니다. |
| **LOG** | `LOG [--sort-by=date\|author]` | 전체 커밋 히스토리를 위상 정렬 순서로 출력하거나, 옵션 지정 시 커스텀 병합 정렬을 활용해 날짜/작성자 기준으로 정렬하여 출력합니다. |
| **SHOW** | `SHOW [commit_hash]` | 지정한 커밋(생략 시 현재 브랜치의 최신 커밋)의 상세 정보(작성자, 날짜, file_meta, 메시지 등)를 출력합니다. |
| **PATH** | `PATH <commit1> <commit2>` | 두 커밋 사이의 무방향 최단 경로를 사전순 Tie-breaking을 적용하여 출력합니다. |
| **ANCESTORS** | `ANCESTORS <commit_hash>` | 스택 기반 반복적 DFS를 통하여 해당 커밋이 상속하고 있는 모든 조상 커밋 목록을 출력합니다. |
| **SEARCH** | `SEARCH <keyword> \| --author=<name>` | 역색인 사전을 활용하여 메시지 키워드 또는 작성자명 기준으로 커밋 목록을 고속 검색합니다. |
| **MERGE** | `MERGE <branch_name>` | 현재 브랜치와 대상 브랜치를 3-Way 병합하여 두 부모를 가지는 Merge 커밋을 자동 조립합니다. 충돌 발생 시 conflict 마커가 삽입됩니다. |
| **USERLIST** | `USERLIST` | 저장소 세션 및 커밋 그래프 내에 등록된 모든 유저 목록과 현재 활성화된 유저를 정렬하여 출력합니다. |
| **EXIT / QUIT** | `EXIT` / `QUIT` | CLI 프로그램을 종료합니다. |

## 6. 제약 사항 및 예외 처리
* 잘못된 파라미터 개수나 형식 대입 시 `Invalid args` 피드백을 출력합니다.
* 저장소가 초기화되기 전(`INIT` 수행 전) 명령을 실행하면 `Error: Repository not initialized.` 에러를 출력하여 비정상 조작을 차단합니다.
* 존재하지 않는 브랜치 접근 시 `Unknown branch: <name>`, 유효하지 않은 커밋 ID 조회 시 `Unknown commit: <hash>` 오류를 띄웁니다.
* 모든 데이터와 커밋 그래프는 세션 단위의 메모리 상에서 안전하게 처리됩니다 (영속성 I/O 및 네트워크 배제).
